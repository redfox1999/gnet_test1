# gnet_flow

基于 gnet v2 的高性能网络框架

## 架构设计
底层是 gnet v2 事件驱动网络库,  在refactor协程里处理数据分包。然后发给worker协程池处事。 Worker 协程里根据CmdID调用对应的业务处理器。
设计有 Worker Pool,用于均衡负载, 避免单个worker协程处理过载。根据 ConnID 分发任务, 避免单个worker协程处理过多任务， 同时让每个连接会在相同的worker协程里处理任务，保证连接的顺序性。


## 特性

- **高性能**：基于 gnet v2 事件驱动网络库
- **Worker Pool**：多 worker 协程池，均衡负载
- **内存池**：分级内存池（128B-64KB），减少 GC 压力
- **连接管理**：原子操作计数，无锁设计
- **日志系统**：基于 zerolog，支持分级、分桶输出
- **协议设计**：固定包头（CmdID 4字节 + DataLen 4字节）+ 可变数据

## 技术栈

- Go 1.21+
- gnet v2
- zerolog

## 快速开始

### 构建

```bash
cd server
make server
```

### 运行

```bash
./bin/server
```

### 配置

编辑 `config/config.yaml`：

```yaml
server:
  addr: ":9000"
  multicore: true
  worker_pool_size: 4
  task_queue_size: 1024

log:
  level: info
  gnet_level: warn
  path: ./logs/
  stdout: true
  filename: server.log
```

### 命令行参数

```bash
./bin/server -version  # 显示版本信息
./bin/server -V        # 显示版本信息（简写）
```

## 协议格式

```
+------------+------------+
|  CmdID(4)  | DataLen(4) |
+------------+------------+
|         Data(N)        |
+-----------------------+
```

- **CmdID**：4字节，命令 ID
- **DataLen**：4字节，数据长度（不含包头）
- **Data**：变长业务数据

## 状态监控

服务器每 5 秒输出状态：

```
📊 状态监控 connections=10000 goroutines=1035 memory=23.28MB pool_get=100000 pool_hit=98500 pool_make=1500
```

- **connections**：当前连接数
- **goroutines**：协程数量
- **memory**：内存占用
- **pool_get**：内存池请求次数
- **pool_hit**：内存池命中次数
- **pool_make**：内存池分配次数

### 调优建议

1. **Worker Pool Size**：根据实际业务场景调整。默认1024
2. **Task Queue Size**：根据业务复杂度调整，建议 1024-4096
3. **内存池**：默认分级桶 128B-64KB，可根据实际包大小调整
